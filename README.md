# Ganzhi2

NSUserTrackingUsageDescription는 iOS 앱이 사용자의 데이터(활동 기록)를 추적하여 광고나 분석 목적으로 활용하려 할 때, 사용자에게 "왜 추적 허용이 필요한지" 설명하는 문구를 정의하는 항목입니다.

현재 Ganzhi 앱의 

Info.plist
에는 다음과 같이 설정되어 있습니다:

문구: "귀하의 취향에 맞는 사주 분석과 맞춤형 광고를 제공하기 위해 활동을 추적합니다."
이 설정이 하는 구체적인 역할은 다음과 같습니다:

1. 개인정보 보호 투명성 (ATT) 준수
Apple은 iOS 14.5부터 App Tracking Transparency (ATT) 정책을 시행하고 있습니다. 앱이 사용자의 기기 식별자(IDFA)에 접근하여 다른 앱이나 웹사이트에서의 활동을 추적하려면 반드시 사용자의 명시적인 동의를 받아야 합니다.

2. 권한 요청 팝업의 본문 표시
앱이 실행될 때(혹은 광고 로드 전) 뜨는 **"앱이 다른 회사의 앱 및 웹사이트에 걸친 사용자의 활동을 추적하도록 허용하시겠습니까?"**라는 팝업 창의 본문 내용으로 이 문구가 사용됩니다.

이 문구가 설정되어 있지 않으면 앱은 사용자 추적 권한을 요청할 수 없으며, 최악의 경우 앱 스토어 심사에서 거절될 수 있습니다.
3. 맞춤형 광고(고단가 광고) 노출
사용자가 '허용'을 누르면 Google AdMob과 같은 광고 네트워크가 사용자의 관심사를 파악할 수 있게 됩니다.

결과: 사용자에게는 더 관련성 높은 광고가 보이고, 앱 개발자에게는 단가가 더 높은 맞춤형 광고가 송출되어 수익 향상에 도움이 됩니다.
요약
NSUserTrackingUsageDescription은 **앱 광고 수익 최적화와 개인정보 보호 정책 준수를 위해 필수적인 '설득 문구'**라고 이해하시면 됩니다. :)
